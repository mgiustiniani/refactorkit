== Business Context

RefactorKit is the system under documentation. It provides deterministic code-intelligence and refactoring capabilities to several kinds of local consumers.

[cols="1,3,2,1",options="header"]
|===
|Actor / neighboring system |Relationship to RefactorKit |Interface |Status

|Developer / Maintainer
|Runs scans, diagnostics, refactorings, recipes, imports, patch apply/rollback, packaging, and tests.
|CLI and optionally library API.
|implementation-informed

|Automation / CI User
|Runs reproducible build, test, golden-test, packaging, and smoke-test workflows.
|CLI and Gradle tasks.
|implementation-informed

|Editor / IDE User
|Uses editor-native navigation, diagnostics, rename, code action, symbols, and semantic token features.
|External editor through LSP.
|implementation-informed

|Local LLM Agent / MCP Client
|Requests focused context and deterministic refactoring/import tools instead of editing files directly.
|MCP / JSON-RPC over stdio.
|implementation-informed

|Embedding JVM Application
|Uses RefactorKit as an in-process library.
|Kotlin/JVM API.
|planned, needs-review for public API stability

|Target Source Workspace
|User-owned source tree containing Java/multi-language files, build metadata, recipes, test data, and `.refactorkit` transaction logs.
|Local filesystem.
|implementation-informed

|Build Toolchain
|Provides Maven/Gradle/JDK/jlink/test execution context.
|Build files, Gradle tasks, shell commands.
|implementation-informed

|External Code Sources
|Provide Java source snippets/files/URLs/LLM responses for safe import preview.
|CLI/MCP input; optional URL/file/clipboard provenance.
|implementation-informed

|External Language Servers
|Future or optional language servers used by the tree-sitter foundation's LSP-backed adapter.
|LSP over stdio.
|planned / stub-informed
|===

== System Boundary

Inside RefactorKit:

* core patch/model/transaction engine;
* Java scanner, symbol model, diagnostics, and refactoring planners;
* CLI application;
* JSON-RPC daemon;
* LSP server;
* MCP server;
* external Java class importer;
* multi-language structural foundation;
* testkit.

Outside RefactorKit:

* user-owned target workspaces;
* editors and IDEs;
* local LLM agents/MCP clients;
* Maven, Gradle, JDK, and `jlink` toolchains;
* external language servers;
* UI frontends and cloud services.
