== Deployment Overview

RefactorKit is a local tool/library, not a hosted service. Current repository evidence supports local process deployment through CLI, daemon, LSP, and MCP applications, plus in-process library consumption by JVM applications.

== Published Deployment Evidence

The immutable `v0.3.0` tag points to commit `ba114134be605b7b0076e7471f8111c081a8aaa3`. The https://github.com/mgiustiniani/refactorkit/releases/tag/v0.3.0[GitHub Release] is latest and non-prerelease and publishes the runtime ZIP with its `.sha256` asset. The downloaded archive matched SHA-256 `0cf054a7eb99a86734f663ca0966da2b47259b8a4658bca97f9044bc2d21ff10`, passed version, help, and sample-project scan checks with `JAVA_HOME` unset, and was deployed locally.

Status: implementation-informed for `v0.3.0`. This does not satisfy the remaining SBOM, provenance/attestation, reproducibility, and stable-contract gates for `v1.0.0`.

== Runtime Dependencies

* Build-time JDK is required for Gradle and `jlink` packaging.
* The packaged CLI is intended not to require a globally installed Java runtime because it contains an embedded runtime.
* Planned authoritative Maven `--release` diagnostics additionally require exact Java platform signatures; the embedded reduced runtime is not those signatures. Source-only/no-release projects instead require an explicitly configured qualified JDK platform or receive typed unavailability.
* Stable assets are promoted from immutable `v1.0.0-rc.N` tags only after the multi-language roadmap and alpha/beta gates complete and independent verification passes; failed tagged candidates are replaced by a new commit and RC number, never by moving a tag.
* Release classification, version metadata, checksum, SBOM, provenance/attestation, and packaged smoke results must agree with the promoted artifact.
* Maven/Gradle projects are scanned from local workspace files.
* Optional external language servers are outside the current deployment and treated as future/optional integrations.

== Missing Deployment Decisions

No evidence was found for:

* cloud/server deployment;
* Docker image or Kubernetes deployment;
* remote daemon exposure over HTTP/WebSocket;
* published artifact repository coordinates beyond the Linux x86_64 stable runtime target;
* OS-specific release matrix beyond the confirmed Linux x86_64 self-contained artifact.

Status: needs-review for later distribution targets; the Linux x86_64 self-contained stable-release target is confirmed by `docs/releases/v1.0.0-plan.md`.
