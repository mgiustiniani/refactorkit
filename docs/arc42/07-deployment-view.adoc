== Deployment Overview

RefactorKit is a local tool/library, not a hosted service. Current repository evidence supports local process deployment through CLI, daemon, LSP, and MCP applications, plus in-process library consumption by JVM applications.

== Runtime Dependencies

* Build-time JDK is required for Gradle and `jlink` packaging.
* The packaged CLI is intended not to require a globally installed Java runtime because it contains an embedded runtime.
* Stable assets are promoted from immutable `v1.0.0-rc.N` tags only after independent verification; failed tagged candidates are replaced by a new commit and RC number, never by moving a tag.
* Release classification, version metadata, checksum, SBOM, provenance/attestation, and packaged smoke results must agree with the promoted artifact.
* Maven/Gradle projects are scanned from local workspace files.
* Optional external language servers are outside the current deployment and treated as future/optional integrations.

== Missing Deployment Decisions

No evidence was found for:

* cloud/server deployment;
* Docker image or Kubernetes deployment;
* remote daemon exposure over HTTP/WebSocket;
* published artifact repository coordinates beyond the Linux x86_64 stable runtime target;
* OS-specific release matrix beyond the confirmed Linux x86_64 self-contained artifact.

Status: needs-review for later distribution targets; the Linux x86_64 self-contained stable-release target is confirmed by `docs/releases/v1.0.0-plan.md`.
