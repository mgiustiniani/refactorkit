== Requirements Overview

RefactorKit is a deterministic refactoring and code-intelligence engine for safe, previewable, rollbackable code transformations.

The first supported language is Java. The Java adapter works with Java 8-era projects while understanding Java language/source constructs and type forms through Java 25. Java is the reference and widest catalogue, while the long-range product target provides equivalent IDE-grade semantic safety and idiomatic depth for TypeScript/JavaScript, the JVM language family, Python, Go, C/C++, C#, Rust, and Clojure without coupling language-specific logic into the core engine.

RefactorKit is intended to replace fragile direct text edits, especially in AI-assisted development workflows, with deterministic semantic operations and patch previews.

The `v0.3.0` foundation release has been published from its immutable tag. Main now develops `0.4.0-SNAPSHOT` as the hardened Java and transaction foundation. Subsequent `0.x` releases build the multi-language kernel and deep language adapters; Clojure is the final planned integration before global `1.0` alpha/beta/RC stabilization. The advertised API remains `0.2` until an explicit migration is accepted. Stable `1.0` means that the core and every advertised language operation have versioned contracts, conservative refusal, IDE-grade idiomatic depth, and all-language acceptance evidence.

Evidence: `README.md`, `AGENTS.md`, `docs/architecture.md`, `docs/releases/v0.4.0-plan.md`, `docs/releases/v1.0.0-plan.md`, ADR 0009.

== Quality Goals

[cols="1,2,2,1",options="header"]
|===
|Quality goal |Meaning for RefactorKit |Evidence |Status

|Safety
|Every code modification is previewable, validated, traceable, and rollbackable.
|`AGENTS.md` safety model; `modules/refactorkit-core` patch and transaction classes.
|implementation-informed

|Determinism
|Refactorings are represented as structured patch plans instead of ad-hoc text replacement.
|`PatchPlan`, `WorkspaceEdit`, `PatchEngine`, Java planner classes.

|Extensibility
|Core is language-independent and language capabilities are provided through adapters.
|`LanguageAdapter` contract; Java adapter and tree-sitter foundation modules.
|implementation-informed

|AI assistance without uncontrolled edits
|Local LLM agents should request focused context and deterministic tools rather than arbitrary rewrites.
|MCP documentation and `docs/ai-workflow.md`.

|Usability across integration modes
|RefactorKit is usable as a library, CLI, daemon, LSP server, and MCP server.
|README, module layout, integration docs.
|implementation-informed

|Runtime portability
|The CLI can be packaged with an embedded Java runtime so end users do not need a globally installed Java runtime.
|`docs/packaging.md`, Gradle jlink tasks.

|Stable and bounded contracts
|Stable surfaces use API version `1.0`, deterministic errors and limits, explicit stable/experimental classification, and compatibility evidence rather than broad feature claims.

|Stable runtime assets are self-contained and accompanied by checksum, SBOM, provenance/attestation, and reproducibility evidence or documented limits.

== Non-Goals and Scope Boundaries

The repository is not intended to implement a custom IDE, Tauri frontend, cloud service, collaborative editing platform, VS Code extension, JetBrains plugin, or web dashboard. Such tools may consume RefactorKit through CLI, JSON-RPC daemon, LSP, MCP, or library APIs, but must not drive core engine design unless they expose stable requirements.
