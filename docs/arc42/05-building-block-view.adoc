Because several modules are libraries rather than independently deployed processes, the C4 model tags them as runtime or test building blocks instead of implying standalone deployment.

== Level 1: RefactorKit System

RefactorKit provides deterministic code-intelligence and refactoring capabilities with preview, validation, apply, and rollback semantics. It exposes those capabilities through CLI, daemon, LSP, MCP, and library integration modes.

== Key Dependencies

* CLI depends on core, Java adapter, web importer, tree-sitter foundation, and testkit.
* Daemon and LSP depend on core and Java adapter.
* MCP depends on core, Java adapter, and web importer.
* Java adapter, web importer, tree-sitter foundation, and testkit depend on core.
* Web importer reuses Java source-root/package/import knowledge.
