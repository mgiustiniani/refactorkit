== Use-case Catalogue

[cols="1,2,2,3,2,1",options="header"]
|===
|ID |Name |Actor |Description |Validation mode |Observed status

|UC-SCAN
|Scan workspace
|Developer, CI, Agent
|Discover workspace files, Java source roots, Maven/Gradle structure, and project snapshot data.
|CLI examples, Java scanner tests, CI smoke scan.
|implementation-informed

|UC-SYMBOLS
|Build/search symbols
|Developer, Agent, Editor
|Expose Java symbols, definitions, and references for focused code intelligence.
|CLI docs, adapter tests, MCP/LSP docs.
|implementation-informed

|UC-DIAGNOSTICS
|Run diagnostics
|Developer, CI, Agent, Editor
|Report structured warnings/errors such as duplicate symbols, package/path mismatch, or file-name mismatch.
|Java adapter tests, integration docs.
|implementation-informed

|UC-PREVIEW-REFACTOR
|Preview refactoring
|Developer, Agent, Editor/App
|Generate `PatchPlan` previews before applying mutating operations.
|Core/Java tests, golden tests, AGENTS safety rules.
|implementation-informed

|UC-APPLY-PATCH
|Apply patch plan
|Developer, Agent/App
|Validate current snapshot and edit safety before writing workspace changes.
|`PatchEngine`, tests.
|implementation-informed

|UC-ROLLBACK
|Rollback transaction
|Developer, Agent/App
|Restore files from transaction rollback metadata after an applied patch.
|Core tests, testkit agent simulation.
|implementation-informed

|UC-JAVA-RENAME-CLASS
|Rename Java class
|Developer, Agent, Editor
|Rename class declaration, file, imports, references, and constructors where supported.
|Golden tests, planner tests, CLI docs.
|implementation-informed

|UC-JAVA-RENAME-MEMBER
|Rename Java method or field
|Developer, Agent, Editor
|Rename scoped Java member occurrences while avoiding unrelated same-name members where possible.
|Planner tests, integration docs.
|implementation-informed

|UC-JAVA-MOVE-CLASS
|Move Java class
|Developer, Agent
|Move class file, update package declaration/imports/FQN references where supported.
|Golden test, planner tests.
|implementation-informed

|UC-JAVA-ORGANIZE-IMPORTS
|Organize Java imports
|Developer, Agent
|Sort/deduplicate imports and remove same-package imports; unused removal remains limited.
|Golden test, docs/testing limitation.
|implementation-informed

|UC-JAVA-SAFE-DELETE
|Safe delete Java symbol
|Developer, Agent
|Refuse deletion when references are found unless explicitly overridden by supported workflow.
|Golden refused case, planner tests.
|implementation-informed

|UC-JAVA-EXTRACT-METHOD
|Extract Java method MVP
|Developer, Agent/Editor
|Extract complete selected lines into no-argument `private void` method for conservative safe cases.
|Extract-method docs and tests.
|implementation-informed

|UC-JAVA-CHANGE-SIGNATURE
|Change Java method signature MVP+
|Developer, Agent/Editor
|Rename/add/reorder/remove parameters for conservative non-overloaded cases.
|Change-signature docs and tests.
|implementation-informed

|UC-RECIPE-RUN
|Run recipe
|Developer, CI, Agent
|Execute YAML-defined Java refactoring workflows with dry-run preview and rollback behavior.
|`docs/recipe-engine.md`, recipe engine tests, sample recipe.
|implementation-informed

|UC-IMPORT-EXTERNAL-JAVA
|Import external Java class
|Developer, Agent
|Assimilate Java source from clipboard/file/URL/snippet/LLM response with provenance/license/conflict checks and preview-only plan.
|Importer docs and tests.
|implementation-informed

|Developer/App/Tool
|Provide long-running local JSON-RPC operations over stdio.
|Daemon module and tests, integration docs.
|implementation-informed

|UC-LSP

|Editor / IDE
|Expose editor-native definition, references, rename, code actions, symbols, semantic tokens, and diagnostics.

|implementation-informed
