== Decision Candidate Coverage

The ADR set covers patch safety, integration-surface and transaction-boundary classification, self-contained verifiable packaging, MCP least privilege, Eclipse JDT evidence-gated semantic authority, offline effective build models/source ownership, and the deferred supreme multi-language `1.0` target. ADRs 0003 and 0007 retain historical context but are superseded by ADR 0008. ADR 0009 supersedes the immediate post-`v0.3.0` Java-focused RC sequence without weakening any accepted safety decision. ADRs 0002 and 0004 record WAL-backed managed transaction guarantees, client-managed native LSP limitations, and the staged recipe-wide transaction boundary without claiming one filesystem-wide ACID commit. ADR 0010 separates package identity from module/source-set ownership and requires a shared Build Model SPI before deep JVM adapter expansion. ADR 0011 makes generic policy allowances ceilings rather than capabilities and requires new provider identities, opaque credential custody, process isolation, provenance, redaction, and native hostile-workspace acceptance before authenticated Maven or executable Gradle support can be claimed. ADR 0012 makes RefactorKit the definitive semantic authority for managed Java operations, fixes the declared full-reactor boundary and release-signature model, and keeps the incomplete diagnostics-authority epic explicitly active/product-critical until its permanent and packaged acceptance passes.
