= ADR 0005: Package the CLI with a Self-Contained jlink Runtime
:toc:
:sectnums:

Date: 2026-07-10

Status: Accepted; stable release evidence pending

== Context

RefactorKit should be easy to run from terminals, CI jobs, editor integrations, and local agent workflows. Requiring users to install or select a compatible global Java runtime increases setup friction and support risk.

The implementation is JVM-based, so a runtime must still be present at execution time.

== Decision

Provide build tasks that assemble a self-contained CLI distribution with an embedded Java runtime generated by `jlink`. The packaged launcher resolves Java from the distribution directory rather than from the system `PATH` or `JAVA_HOME`.

For stable releases, the runtime archive is a versioned product contract. Publish its checksum, SBOM, and provenance or attestation, and document verification commands. Compare clean package builds, remove avoidable nondeterminism, and document any remaining reproducibility limits. Verify archive layout, executable permissions, embedded runtime, license files, version metadata, and an execution smoke test with `JAVA_HOME` unset before promoting an immutable release candidate to `v1.0.0`.

== Consequences

* Users can run the packaged CLI without installing Java globally.
* Distribution artifacts are larger than plain application archives.
* jlink packaging is OS-specific, so release automation must build per target platform.
* A JDK remains required at build time to create the runtime image.
* Runtime module selection must be maintained as dependencies evolve.
* Release automation must classify prerelease and stable artifacts from immutable tags and must not move or recreate a failed tag.
* Self-containment, checksum, SBOM, provenance, and reproducibility claims require evidence from the published artifact; successful assembly alone is insufficient.
