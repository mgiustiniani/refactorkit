= ADR 0003: Start Java Refactoring with a Lexical MVP Before Compiler-Backed Analysis
:toc:
:sectnums:

Date: 2026-07-10

Status: Superseded by ADR 0008

== Context

RefactorKit must support early Java refactoring workflows before a full compiler-backed semantic model is available. Full binding-aware Java analysis is valuable, but integrating and hardening it would delay alpha feedback.

The alpha still needs conservative scanning, symbol discovery, reference search, and initial refactoring previews.

== Decision

For the historical alpha, use a lexical and structural Java MVP. Implement package, class, member, import, and reference detection conservatively, and refuse or warn for ambiguous cases. Compiler-backed binding analysis was not an alpha prerequisite.

ADR 0007 subsequently required compiler/classpath-aware evidence for semantic guarantees, and ADR 0008 selected Eclipse JDT as the primary analysis strategy. This ADR therefore no longer authorizes lexical analysis to act as hidden semantic authority for stable operations.

== Consequences

* Alpha users could exercise the end-to-end refactoring workflow earlier.
* Conservative warnings remain necessary for reflection, framework strings, generated code, overloads, and other semantic risks.
* The Java adapter must avoid claiming certainty when lexical evidence is insufficient.
* Compiler-backed analysis can replace or augment lexical detection without changing the PatchPlan safety model.
* Documentation must present the alpha as useful but not equivalent to a mature IDE semantic engine.
