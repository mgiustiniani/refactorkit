# v0.7.0 TypeScript/JavaScript IDE diagnostics acceptance

Status: implementation acceptance for additive API `0.2` `diagnostics.v2`.
This is not a release tag or a retroactive `v0.6.2` capability claim.

## Reference reproduction

Immutable reference:

```text
tag: v0.6.2
commit: 110b60537009781b44db9b840cf057d5b7f51067
```

Validated from a detached worktree:

```bash
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
./gradlew :modules:refactorkit-daemon:installDist --no-daemon
# BUILD SUCCESSFUL

unset JAVA_HOME # PATH selects GraalVM JDK 25.0.3
./gradlew clean :modules:refactorkit-daemon:installDist --no-daemon
# fails in Kotlin 2.0.21 JavaVersion.parse("25.0.3")
```

Source inspection confirmed the legacy bare array, discarded character/end range,
missing authority envelope, LSP capability/runtime mismatch, and flattened
compiler-unavailable diagnostics. `main` had not changed those paths before this
slice. Detailed classification is in
[`../typescript-diagnostics-protocol.md`](../typescript-diagnostics-protocol.md).

## Local acceptance

Commands:

```bash
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH="$JAVA_HOME/bin:$PATH"
./gradlew clean build goldenTest --no-daemon
./gradlew :modules:refactorkit-cli:refactorkitRuntimeZip \
  :modules:refactorkit-cli:smokePackagedCli --no-daemon
python3 scripts/smoke-packaged-typescript.py
python3 scripts/smoke-packaged-kill-recovery.py
```

Accepted evidence:

- 567 JVM tests pass with zero failures, errors, or skips;
- all 23 golden cases pass;
- packaged CLI/daemon and transaction smoke pass;
- real pinned Node 22.18.0, `typescript-language-server` 5.1.3, and TypeScript
  5.9.3 prove capability metadata, saved-disk envelope, immutable unsaved overlay,
  exact UTF-16 ranges, compiler process attestation, response byte attestation,
  stale lease refusal, legacy diagnostics, semantic restart, managed apply, WAL,
  rollback, and kill recovery;
- source images remain unchanged after overlay diagnostics;
- JDK 25 now fails during Gradle configuration with the actionable JDK 21
  prerequisite instead of an internal Kotlin compiler stack trace.

Native CI must run the same packaged TypeScript qualification on Linux x86-64,
Windows x86-64, macOS x86-64, and macOS arm64 before the fix is reported complete.

## Compatibility

- `diagnostics` remains the legacy bare array.
- `diagnostics.v2` is additive and has response schema version 2.
- `typescript.semantic.start/restart` add `semanticLease` and `snapshotHash` fields.
- capability schema version remains 1 and gains optional diagnostics-only fields;
  existing consumers that tolerate additive object fields remain compatible.
- CLI adds `typescript diagnostics-v2`; MCP adds `diagnostics_v2`.

## Remaining unsupported boundaries

- editor overlays may replace only existing TypeScript/JavaScript source files;
- new unsaved files, tsconfig/package overlays, and mixed-language overlay batches
  are refused;
- LSP publication diagnostics are not accepted as exact compiler authority;
- `line` and `none` locations cannot be converted to Monaco ranges;
- compiler heap is bounded by V8 old space, not by an OS RSS sandbox;
- JDK 25 source builds remain unsupported until the build toolchain is upgraded
  and separately qualified.
