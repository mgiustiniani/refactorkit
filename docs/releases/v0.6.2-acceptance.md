# RefactorKit v0.6.2 Acceptance Record

Status: published and independently verified.

Date: 2026-07-14.

## Purpose

`v0.6.2` is the publication patch for the completed `v0.6.0` feature scope. The
immutable `v0.6.0` and `v0.6.1` tags were not published because their Windows tag
jobs failed before packaging; no partial release was created and neither tag was
moved. Their pre-tag branch CI runs had reported all four native targets green,
but cached test tasks had not exposed the clean-build defect.

The failed `v0.6.1` tag exposed host-dependent validation that passed glob syntax
to `java.nio.file.Path`: Windows correctly rejected `src/**/*.ts`. `v0.6.2`
validates include/exclude and path-alias patterns lexically, preserving workspace
escape refusal without asking the host path parser to interpret wildcards. A
bounded reference request now completes the semantic project barrier before
managed rename so lazy Windows projects cannot return declaration-only edits. The
release workflow retains `clean build --no-daemon`, followed by a separate
package/native-smoke command. API `0.2`, qualified toolchain and safety gates are
unchanged.
