# RefactorKit v0.7.x support matrix

Status: active `0.7.0-SNAPSHOT` qualification ledger. Rows marked **pending native**
are not release claims until their four-platform workflow succeeds.

## Runtime assets

| Host | Architecture | Embedded Java runtime | Native managed acceptance |
|---|---|---:|---|
| Linux | x86_64 | yes | yes |
| Windows | x86_64 | yes | yes |
| macOS | Intel x86_64 | yes | yes |
| macOS | Apple Silicon arm64 | yes | yes |

The package requires no globally installed Java. Historical Java platform analysis
uses the packaged/explicitly attested `ct.sym`; the reduced runtime includes
`jdk.zipfs`. macOS and Windows packages remain unsigned.

## Java managed refactoring catalogue

| Operation | Contract status | Qualified boundary |
|---|---|---|
| `renameClass` / `renameMember` | beta-contract | exact JDT declaration/reference binding; framework/generated/dynamic boundaries remain conservative |
| `moveClass` / package migration | beta-contract | package/import/FQCN rewrite with preview, diagnostics, WAL and rollback |
| `moveSourceRoot` | beta-contract | complete root, unchanged source bytes/package/FQCN; no POM edits |
| `java.moveAcrossMavenModules` | experimental, pending native | complete non-generated root between existing Maven modules plus exact literal consumer dependency rewrite; one Java/POM transaction |
| `changeSignature.*` | experimental bounded catalogue | exact selected methods, hierarchy/constructor/functional-reference rows documented separately |
| `extractMethod` | experimental limited | conservative control/data-flow refusal |
| `organizeImports` / `formatFile` | beta-contract | generated or ambiguous inputs refuse |
| `safeDelete` | beta-contract | references refuse unless explicit force policy allows the row |

Every managed write requires preview, explicit authorization, exact staged
diagnostics, `PatchEngine`, WAL, recovery metadata and rollback. Native editor-
applied LSP edits remain client-managed and do not advertise RefactorKit rollback.

## Maven ownership migration limits

The first `java.moveAcrossMavenModules` row supports existing reactor modules and
literal jar/no-classifier module identities. It preserves POM bytes outside exact
coordinate text ranges and rejects property/profile/inherited ambiguity, duplicate
origins without explicit all-occurrence intent, cycles, missing destination edges,
remaining source dependencies, generated code and diagnostics regressions.

Module-directory moves, parent `<modules>` rewrites and group/artifact/version
migration are not yet qualified. They must remain explicit future arguments and
must not be inferred from Java package moves.

## Framework and dynamic-risk evidence

Spring, Jakarta/JPA and Jackson annotations are detected without loading framework
dependencies. Relevant rename/move/delete/ownership operations elevate risk and
report deterministic annotation locations. Exact quoted moved FQCN candidates
are reported for runtime/configuration review. RefactorKit does not claim to
rewrite reflection, XML configuration, persisted names, JPQL, serialized payloads
or generated metamodel/processor output.

## Integration compatibility

API version remains `0.2`. Existing CLI commands, daemon methods, MCP tools and
beta operation envelopes remain supported. New diagnostics/index fields are
additive. `java.moveAcrossMavenModules` is inventoried as an additive experimental
operation carried by the existing preview/apply methods and tools.
