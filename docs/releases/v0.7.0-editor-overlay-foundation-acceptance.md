# v0.7.0 Immutable Editor Overlay Foundation Acceptance

Status: accepted core foundation and first TypeScript intelligence routing row.

## Qualified boundary

`refactorkit-core` now owns the language-neutral immutable editor overlay model.
An overlay is derived from one exact saved `ProjectSnapshot` and contains one or
more existing versioned source documents. It creates a provider snapshot entirely
in memory and never modifies the saved snapshot or workspace filesystem.

Each document binds:

- normalized workspace-relative protocol path;
- non-negative editor document version;
- exact content and SHA-256 authority;
- language inherited from the saved source;
- exact base and derived provider snapshot hashes.

The aggregate binds a deterministic overlay hash, sorted document metadata and
bounded UTF-8 size. `SemanticQueryEnvelope` rejects an overlay from another base
snapshot or document authority whose path, version or content hash differs.

## Validation and limits

| Boundary | Behavior |
|---|---|
| Documents | 1–128 |
| Aggregate content | 786,432 UTF-8 bytes |
| Per-document content | Existing 10 MiB source-file limit |
| Path | Existing saved source, relative `/` protocol form, no traversal/drive/backslash |
| Language | Must match the requested provider language when supplied |
| Version | Non-negative and exact |
| Content | NUL rejected |
| Duplicate path | Rejected before provider snapshot creation |

## Compatibility migration

TypeScript/JavaScript API `0.2` `diagnostics.v2` now derives its
`immutable-editor-overlay` provider snapshot through the shared core model.
Request and response JSON remain unchanged, including overlay hash, document
version/content hashes, provider snapshot hash, exact diagnostics and refusal
codes.

## First intelligence routing row

`intelligence.query` accepts `immutable-editor-overlay` authority for
TypeScript/JavaScript `documentSymbols`. It requires the exact saved snapshot,
index generation, semantic lease, selected overlay path and monotonic document
version. The provider temporarily opens overlay documents, returns exact UTF-16
symbols and restores saved documents before returning. Responses expose only
path/version/content hashes, overlay/provider snapshot hashes and provenance;
source content is never echoed.

TypeScript/JavaScript completion, hover and signature help now use the same
authority and restoration pipeline. Completion returns bounded typed candidates, insertion and
replacement data, additional edits and explicit incompleteness. Hover returns
bounded plaintext/Markdown sections and an optional exact UTF-16 range.
Signature help returns bounded overloads, exact parameter-label spans, and active
signature/parameter state. Other language providers remain refused until their
persistent semantic implementations qualify.

Interactive daemon requests are now asynchronously queued behind FIFO stateful
barriers. Queries inside one barrier segment use interactive/normal/background
priority, and `intelligence.cancel` cooperatively cancels queued or running work.
Acknowledged LSP cancellation restores saved documents and preserves the provider;
unacknowledged cancellation fails closed by stopping it.
