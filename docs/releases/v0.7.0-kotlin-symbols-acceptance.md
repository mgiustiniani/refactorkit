# v0.7.0 Kotlin JVM declaration-symbol acceptance

Status: cross-platform implementation acceptance for experimental saved-snapshot
declared-type/function symbol search and ID-based definition. Native qualification
passed in GitHub Actions run `29418213082`. This does not qualify references, a
Kotlin mutation, or a release tag.

## Qualified row

| JDK | Kotlin compiler | Build projection | Capabilities |
|---|---|---|---|
| 21 | `kotlin-compiler-embeddable`/`kotlin-stdlib` 2.0.21, annotations 13.0 | `kotlin-jvm-projection-v1` `AVAILABLE`, one Kotlin source module | `workspaceSymbols`, `documentSymbols`, `definition` `EXPERIMENTAL/COMPILER/NONE` for classes, named/companion objects, interfaces, enum classes, annotation classes and bounded non-overloaded functions |

Backend: `kotlin-compiler-jvm-declarations-k2-v1`, replacing the pre-release
regular-type-only identity `kotlin-compiler-jvm-types-k2-v1`. The authoritative
daemon response schema is `docs/api-0.2-kotlin-symbols-schema.json`.

## Authority boundary

The worker first compiles the complete bounded source image with K2. Symbols are
available only when compilation succeeds. It then creates compiler PSI from the
same immutable overlay and accepts a declaration only when a generated class file
proves the corresponding JVM binary name. Type IDs are
`kotlin-jvm-type-v1:<sha256>` values derived from that binary name. Function IDs are `kotlin-jvm-callable-v1:<sha256>` values derived from the
compiler file-facade/containing owner plus the exact non-synthetic class-file JVM
method name and descriptor. Exact name
ranges come from compiler PSI and are validated against UTF-16 source content in
the attested snapshot.

The catalogue contains non-local top-level and nested classes, named/data/nested
objects, companion objects, interfaces, enum classes and annotation classes.
Enum entries and anonymous object expressions are not symbols. An unsupported JVM
name, duplicate identity, missing class file, malformed range, result overflow or
compiler error refuses the whole declaration result. The first function row
requires exactly one generated non-synthetic/non-bridge JVM method for a source
name; overloads, bridge ambiguity and `@JvmName` refuse rather than guessing.
Properties, constructors, type aliases and local/anonymous classes/functions are
outside the catalogue. Implicit companions use
compiler name `Companion` and select the exact `object` keyword because there is
no source name identifier. The matching
stdlib and qualified annotations runtime are explicit attested compiler inputs;
missing runtime evidence refuses before process execution. There is no lexical fallback or partial declared-type index.

## Local and packaged acceptance

```bash
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH="$JAVA_HOME/bin:$PATH"
./gradlew clean build goldenTest --no-daemon
./gradlew :modules:refactorkit-cli:refactorkitRuntimeDist --no-daemon
python3 scripts/smoke-packaged-kotlin.py
```

Accepted behavior:

- 635 JVM tests pass locally; the existing 23 golden cases remain unchanged by
  this read-only declaration-catalogue slice;
- real K2 compilation returns committed class, object, interface, enum-class and
  annotation-class declarations, including data/nested/companion objects while
  excluding an anonymous object expression;
- opaque type and callable IDs are stable across source-offset shifts and separate
  symbol/definition invocations; callable IDs change when the JVM descriptor
  changes;
- extension, suspend, erased-generic and default-argument functions publish only
  when one exact primary generated method proves the identity;
- definition returns exact zero-based UTF-16 declaration ranges, including an
  implicit companion-object definition over its `object` keyword and a member
  function definition over its compiler PSI name;
- snapshot, build, toolchain and external-process attestations are present;
- the ready envelope validates against the Draft 2020-12 Kotlin symbols schema;
- stale leases and snapshots refuse before compiler authority;
- broken compilation returns `kotlin.symbolCompilationFailed`, not a lexical
  approximation;
- malformed worker fields, paths, ranges, duplicate IDs and unsupported results
  fail closed;
- source bytes remain unchanged after symbols and definition;
- GitHub Actions run `29418213082` passes the Linux x86-64 build/packaged Kotlin
  qualification plus packaged native runtime jobs on Windows x86-64, macOS x86-64
  and macOS arm64.

## Limits

- the diagnostics worker limits remain active: one process, 30 seconds, 512 MiB
  heap, 8 MiB stdout, 64 KiB stderr, 96 Kotlin source files and 128 classpath
  entries;
- at most 500 compiler symbols may be extracted;
- daemon/CLI/MCP protocol output returns at most 200 symbols with explicit
  truncation metadata;
- declaration names are at most 512 UTF-16 code units, callable JVM descriptors
  at most 1,024 characters, and each generated callable-owner class at most
  16 MiB and 10,000 non-synthetic/non-bridge methods;
- only saved-disk snapshots are authoritative.

## Explicitly unsupported

- type aliases;
- overloaded/bridge-ambiguous/`@JvmName` functions, constructors, properties, parameters
  and type parameters;
- local/anonymous classes, local functions, lambdas and anonymous object expressions;
- backtick/non-ASCII JVM declaration-name shapes in this first row;
- resolving a definition from a usage location;
- references and Java/Kotlin cross-language identity;
- `.kts`, Android, Multiplatform, generated/plugin-derived semantics and
  immutable editor overlays;
- every Kotlin mutation.

Because these capabilities are read-only, `PatchEngine`, WAL and rollback are not
entered. The returned symbol ID is navigation evidence only and must not be used
as rename authority. Requirements are recorded in
`docs/requirements/kotlin-jvm-declared-type-symbols.md` and
`docs/requirements/kotlin-jvm-object-symbols.md` and
`docs/requirements/kotlin-jvm-function-symbols.md`.
