# v0.7.0 Kotlin JVM type-symbol acceptance

Status: local implementation acceptance for experimental saved-snapshot
declared-type symbol search and ID-based definition. Cross-platform native
qualification of the expanded kind row remains pending. This does not qualify
references, a Kotlin mutation, or a release tag.

## Qualified row

| JDK | Kotlin compiler | Build projection | Capabilities |
|---|---|---|---|
| 21 | `kotlin-compiler-embeddable`/`kotlin-stdlib` 2.0.21, annotations 13.0 | `kotlin-jvm-projection-v1` `AVAILABLE`, one Kotlin source module | `workspaceSymbols`, `documentSymbols`, `definition` `EXPERIMENTAL/COMPILER/NONE` for classes, interfaces, enum classes and annotation classes |

Backend: `kotlin-compiler-jvm-types-k2-v1`. The authoritative daemon response
schema is `docs/api-0.2-kotlin-symbols-schema.json`.

## Authority boundary

The worker first compiles the complete bounded source image with K2. Symbols are
available only when compilation succeeds. It then creates compiler PSI from the
same immutable overlay and accepts a declaration only when a generated class file
proves the corresponding JVM binary name. Public IDs are
`kotlin-jvm-type-v1:<sha256>` values derived from that binary name. Exact name
ranges come from compiler PSI and are validated against UTF-16 source content in
the attested snapshot.

The catalogue contains non-local top-level and nested classes, interfaces, enum
classes and annotation classes. Enum entries are not symbols. An unsupported
class-like declaration such as an object, an unsupported JVM-name shape,
duplicate identity, missing class file, malformed range, result overflow or
compiler error refuses the whole declared-type result. Callables, type aliases,
local classes and anonymous classes are outside the catalogue. The matching
stdlib and qualified annotations runtime are explicit attested compiler inputs;
missing runtime evidence refuses before process execution. There is no lexical fallback or partial declared-type index.

## Local and packaged acceptance

```bash
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH="$JAVA_HOME/bin:$PATH"
./gradlew clean build goldenTest --no-daemon
./gradlew :modules:refactorkit-cli:refactorkitRuntimeDist --no-daemon
python3 scripts/smoke-packaged-kotlin.py
```

Accepted behavior:

- 630 JVM tests pass locally; the existing 23 golden cases remain unchanged by
  this read-only type-catalogue slice;
- real K2 compilation returns committed class, interface, enum-class and
  annotation-class declarations;
- opaque IDs are stable across source-offset shifts and separate symbol/definition
  invocations for the same saved snapshot;
- definition returns exact zero-based UTF-16 name ranges, including a non-class
  interface definition;
- snapshot, build, toolchain and external-process attestations are present;
- the ready envelope validates against the Draft 2020-12 Kotlin symbols schema;
- stale leases and snapshots refuse before compiler authority;
- broken compilation returns `kotlin.symbolCompilationFailed`, not a lexical
  approximation;
- malformed worker fields, paths, ranges, duplicate IDs and unsupported results
  fail closed;
- source bytes remain unchanged after symbols and definition;
- the expanded kind row must still run on Linux x86-64, Windows x86-64, macOS
  x86-64 and macOS arm64 before cross-platform qualification is reported complete.

## Limits

- the diagnostics worker limits remain active: one process, 30 seconds, 512 MiB
  heap, 8 MiB stdout, 64 KiB stderr, 96 Kotlin source files and 128 classpath
  entries;
- at most 500 compiler symbols may be extracted;
- daemon/CLI/MCP protocol output returns at most 200 symbols with explicit
  truncation metadata;
- declaration names are at most 512 UTF-16 code units;
- only saved-disk snapshots are authoritative.

## Explicitly unsupported

- objects, companion objects and type aliases;
- functions, constructors, properties, parameters and type parameters;
- local and anonymous classes;
- backtick/non-ASCII JVM declaration-name shapes in this first row;
- resolving a definition from a usage location;
- references and Java/Kotlin cross-language identity;
- `.kts`, Android, Multiplatform, generated/plugin-derived semantics and
  immutable editor overlays;
- every Kotlin mutation.

Because these capabilities are read-only, `PatchEngine`, WAL and rollback are not
entered. The returned symbol ID is navigation evidence only and must not be used
as rename authority. Requirements are recorded in
`docs/requirements/kotlin-jvm-declared-type-symbols.md`.
