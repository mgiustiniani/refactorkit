# RefactorKit v0.6.x support matrix

Status: `v0.6.2` publication contract for the completed `v0.6.0` feature scope.

## Runtime assets

| Host | Architecture | Asset | Native managed acceptance |
|---|---|---|---|
| Linux | x86_64 | `refactorkit-runtime-0.6.2-linux-x86_64.zip` | yes |
| Windows | x86_64 | `refactorkit-runtime-0.6.2-windows-x86_64.zip` | yes |
| macOS | Intel x86_64 | `refactorkit-runtime-0.6.2-macos-x86_64.zip` | yes |
| macOS | Apple Silicon arm64 | `refactorkit-runtime-0.6.2-macos-aarch64.zip` | yes |

Every asset embeds Java. TypeScript semantic operations additionally require the
explicit external toolchain below; it is qualification input and is not bundled.

## TypeScript semantic toolchain

| Component | Managed supported version | Discovery boundary |
|---|---|---|
| Node.js | `22.18.0` | majors 18–24 parse as experimental candidates |
| `typescript-language-server` | `5.1.3` | another strict version requires new qualification evidence |
| TypeScript | `5.9.3` | major 5 parses as an experimental candidate |

Only the exact managed row is a `v0.6.0` support claim. CI acquires the two npm
packages from the public registry using the committed lockfile and
`npm ci --ignore-scripts --no-audit --no-fund`. Runtime discovery never executes
npm, package scripts, project scripts, bundlers, plugins, or task runners.
Both qualified npm packages declare Apache-2.0. Their integrity hashes are in
`qualification/typescript-toolchain/package-lock.json`.

## Language capability

| Language/surface | Read operations | Managed mutation | Boundary |
|---|---|---|---|
| Java | existing `0.5` JDT-backed catalogue | existing preview/apply/WAL/rollback | adapter reference implementation |
| TypeScript `.ts`/`.tsx` | compiler/LSP symbols, definition, references, exact compiler diagnostics | identifier `renameSymbol` through CLI, daemon and MCP | experimental capability; explicit toolchain required |
| Checked JavaScript | semantic reads and proposal generation | eligible only when completeness is `CHECKED_JAVASCRIPT` | explicit toolchain and exact compiler gate required |
| Dynamic or mixed JavaScript | bounded semantic/structural reads | proposal-only | managed gate refuses incomplete semantics |
| RefactorKit LSP for TypeScript/JavaScript | structural `.ts`/`.js` document symbols | editor/native language client owns standard LSP edits | no RefactorKit WAL for editor-applied native LSP edits |

TypeScript managed rename is qualified for exact identifiers, path aliases and
re-exports. Advanced move/file rename, organize imports, change signature,
extract/inline and recipes are deferred to `v0.7.x`.

### IDE diagnostics protocol boundary

The immutable `v0.6.2` daemon exposes exact compiler diagnostics internally but
its legacy `diagnostics` wire is a bare array with one-based line only and no
snapshot/session/source authority. It is supported for legacy Problems-style
consumers, not exact Monaco markers or unsaved buffers. Additive API `0.2`
`diagnostics.v2` is first available on the `0.7.0-SNAPSHOT` line; see
`docs/typescript-diagnostics-protocol.md`. No `v0.6.2` asset is retroactively
claimed to contain that contract.

## Safety and resource boundary

Qualified packaged acceptance covers:

- exact before/staged compiler diagnostics from a source-only overlay;
- explicit approval, `PatchEngine`, WAL, rollback and startup recovery;
- forced language-server termination and provenance-preserving bounded restart;
- daemon kill after durable journal state `APPLYING` during a 111-file rename,
  followed by exact compensation to `ROLLED_BACK`;
- symlink, malformed metadata, path, stream, frame, document, result, timeout and
  process-tree limits.

The 512 MiB flag limits V8 old space, not total operating-system RSS. External
semantic processes are controlled proposal providers, not filesystem/network
sandboxes. macOS and Windows assets are unsigned; possible Gatekeeper/SmartScreen
warnings are tracked as Arc42 debt `TD-DIST-001`.
