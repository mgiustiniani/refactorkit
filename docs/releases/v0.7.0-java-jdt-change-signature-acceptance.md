# v0.7.0 Java JDT change-signature acceptance

Status: bounded JDT parameter rename/type-change/add/remove/reorder are qualified
through packaged CLI/daemon/MCP execution on all four native platforms.

## Qualified behavior

The planner resolves an exact JDT method binding, including a signed overload
selector, and publishes one parameter binding with exact declaration/use ranges.
Only those ranges are edited. Same-name fields, other overloads, comments and
literals remain unchanged.

Staged JDT analysis must re-establish method signature, parameter index, new name
and usage count. Introduced error diagnostics refuse while unchanged baseline
errors remain tolerated. Preview remains read-only; apply and rollback use the
normal `PatchEngine` and WAL transaction path.

## Local evidence

- 694 JVM tests, zero failures/errors/skips;
- exact parameter declaration plus two bound-use extraction;
- signed overload selection and same-name field/overload exclusion;
- malformed, unsigned-overload and missing-parameter refusal coverage;
- packaged CLI, daemon and MCP preview, apply and byte-identical rollback using
  the embedded JDK;
- full existing golden suite remains green.

## Native evidence

GitHub Actions run `29607477633` passed packaged parameter rename on Linux
x86-64, Windows x86-64, macOS x86-64 and macOS arm64. Run `29614333900`

## Expanded local rows

JDT now publishes exact method parameter-list and bound invocation argument-list
ranges. Add/remove/reorder update only the selected signed overload and its JDT
call sites, then recompile a disposable staged source overlay. Public/protected,
interface, override, method-reference and incomplete shapes refuse.

## Parameter type-change local row

One exact regular parameter type token can change when the method is
package/private and outside hierarchies. Staged JDT must retain every original
call-site location and parameter body-use count. Incompatible callers/bodies,
overload drift, annotations, arrays and varargs refuse. CLI/daemon/MCP apply and
rollback restore exact bytes locally.

## Override-family local row

`addParameter` with `includeHierarchy=true` and explicit external-consumer-risk
acceptance updates every JDT-connected source interface/override declaration and
all calls bound to the family. Disposable staged JDT must retain a connected
family and per-file call counts. Missing external declarations refuse. Local
packaged CLI/daemon/MCP apply and exact rollback pass.

Hierarchy remove-parameter passes across declarations with different parameter
names on all four native platforms (run `29624918322`): the selected index must be unused in every body, and the same
argument index is removed from all family calls. Packaged CLI/daemon/MCP exact rollback passes.

Hierarchy reorder passes local library acceptance. The selected declaration's
requested names become an index permutation applied to every family declaration
and bound call, so implementation parameter names may differ.
