# v0.7.0 Java JDT change-signature acceptance

Status: bounded parameter-rename library and packaged CLI row complete locally;
four-platform native qualification and daemon/MCP-specific acceptance remain
pending.

## Qualified behavior

The planner resolves an exact JDT method binding, including a signed overload
selector, and publishes one parameter binding with exact declaration/use ranges.
Only those ranges are edited. Same-name fields, other overloads, comments and
literals remain unchanged.

Staged JDT analysis must re-establish method signature, parameter index, new name
and usage count. Introduced error diagnostics refuse while unchanged baseline
errors remain tolerated. Preview remains read-only; apply and rollback use the
normal `PatchEngine` and WAL transaction path.

## Local evidence

- 683 JVM tests, zero failures/errors/skips;
- exact parameter declaration plus two bound-use extraction;
- signed overload selection and same-name field/overload exclusion;
- malformed, unsigned-overload and missing-parameter refusal coverage;
- packaged CLI preview, apply and byte-identical rollback using the embedded JDK;
- full existing golden suite remains green.
