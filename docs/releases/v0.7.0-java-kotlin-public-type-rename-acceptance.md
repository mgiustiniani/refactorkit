# v0.7.0 Kotlin-to-Java public type rename acceptance

Status: local packaged acceptance complete; four-platform native qualification
pending.

## Implemented row

A public top-level Kotlin type in one mixed JVM module can be previewed when:

- K2 compilation of the exact immutable snapshot succeeds;
- the declaration exposes one compiler/class-file-proven JVM binary name;
- the generated classes remain inside the disposable compiler overlay;
- JDT resolves Java uses from that ephemeral output and reports the same qualified
  binary identity with exact source ranges;
- Kotlin usages are exact K2 FIR/PSI evidence;
- the caller explicitly sets `acceptExternalConsumerRisk=true` (CLI:
  `--accept-external-consumer-risk`);
- before and staged K2/JDT diagnostics are clean;
- no dynamic quoted name, generated source, collision or incomplete identity is
  present.

The normalized plan edits only exact Kotlin and Java tokens. Daemon and MCP apply
use a combined `kotlin-k2-java-jdt` diagnostics gate. All edits pass through one
`PatchEngine` transaction with WAL and rollback.

## Remaining gate

The packaged smoke must pass on Linux x86-64, Windows x86-64, macOS x86-64 and
macOS arm64. The symmetric public Java type used by Kotlin remains a separate
mandatory requirement row and is not implied by this acceptance.
