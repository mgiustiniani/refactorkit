# v0.7.0 Kotlin compiler diagnostics acceptance

Status: implementation acceptance for experimental read-only diagnostics. This
does not qualify a Kotlin mutation or authorize a release tag.

## Qualified row

| JDK | Kotlin compiler | Build projection | Capability |
|---|---|---|---|
| 21 | `kotlin-compiler-embeddable` 2.0.21 | `kotlin-jvm-projection-v1` `AVAILABLE`, one Kotlin source module | `diagnostics` `EXPERIMENTAL/COMPILER/NONE` |

Backend: `kotlin-compiler-diagnostics-k2-v1`.

## Local acceptance

```bash
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH="$JAVA_HOME/bin:$PATH"
./gradlew clean build goldenTest --no-daemon
./gradlew :modules:refactorkit-cli:installDist --no-daemon
modules/refactorkit-cli/build/install/refactorkit/bin/refactorkit \
  kotlin diagnostics samples/kotlin-maven-simple \
  --jdk-home "$JAVA_HOME" \
  --compiler-jar <kotlin-compiler-embeddable-2.0.21.jar> \
  --compiler-classpath <explicit-runtime-classpath> \
  --request-id packaged-kotlin
```

Accepted behavior:

- 575 JVM tests and all 23 golden cases pass locally;
- a real K2 invocation returns `ready` and no diagnostics for the committed
  `samples/kotlin-maven-simple` source;
- unresolved Kotlin types return compiler diagnostics associated with the exact
  workspace-relative source and explicit `LINE_ONLY` precision;
- source bytes are unchanged after compiler execution;
- toolchain, build projection and snapshot hashes plus process executable/argument
  hashes are returned;
- toolchain drift, scripts, partial models, unsupported platforms, stale leases,
  stale snapshots, unsafe/missing classpaths and limits refuse before authority is
  granted;
- compiler plugins and arbitrary/script arguments are rejected by the isolated
  bridge before compiler class loading;
- malformed/incomplete worker output, internal compiler failure, timeout, output
  overflow and overlay mutation are typed errors rather than source diagnostics;
- discovery and startup execute no Maven/Gradle/plugin/compiler input.

Native CI must pass the real compiler tests on Linux x86-64, Windows x86-64,
macOS x86-64 and macOS arm64 before this slice is reported complete.

## Limits

- 30-second request timeout;
- one compiler process;
- 512 MiB compiler heap;
- 8 MiB stdout and 64 KiB stderr;
- 6 MiB captured compiler XML;
- 96 Kotlin source files;
- 128 project classpath entries;
- 500 diagnostics and 4,096 characters per message.

## Explicitly unsupported

- `.kts` script semantics;
- Gradle/Maven models that remain `PARTIAL` or execution-refused;
- Android, Multiplatform and `expect`/`actual`;
- compiler plugins, kapt, KSP, annotation processors and javac integration;
- workspace-local compiled classpath outputs;
- immutable editor overlays;
- Kotlin symbols, definition, references and Java/Kotlin semantic identity;
- every Kotlin refactoring or mutation.

Because this capability is read-only, `PatchEngine`, WAL and rollback are not
entered. Those gates remain mandatory for the first Kotlin mutation.
