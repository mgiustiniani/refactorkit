# RefactorKit 0.7.0 authoritative Java platform/JDT acceptance

Status: first local and packaged Java 8/21 row complete on all four native
platforms; full release matrix remains open.

## Qualified boundary

For an effective Maven `maven.compiler.release`, the Maven build model publishes
`java.release` and `java.platformSelection=maven-release` separately from source
syntax level. `JdtJavaSemanticAnalyzer.analyzeAuthoritatively` requires one
matching `JavaReleasePlatformAuthority` and re-attests it before and after
analysis. Historical releases project only that release's bounded `ct.sym`
signatures into a disposable classpath which is deleted after use. The current
release binds a complete `java.se` module image and explicit `jrt-fs.jar`.

CLI, daemon and MCP preserve the full declared reactor while an optional module
selector filters results afterward. A source-only configuration publishes
`java.platformSelection=explicit-jdk-required` and receives typed refusal. The
host JVM and packaged runtime are not used as implicit platform evidence.

This row does not yet promote Java/Maven diagnostics or mutation authority.
Classpath/source-set completeness, full-reactor impact closure, provider
availability and the releases 8-through-25 packaged matrix remain required.

## RED/GREEN evidence

The permanent release-8 adapter scenario initially failed with `diagnostics=[]`,
proving the previous compliance-only path accepted newer APIs. The first packaged
scenario initially returned `java.platform.classpathProjectionFailed` for both
release-21 source sets because `ct.sym` represents its current release with a
system-module marker rather than historical `.sig` entries. The green correction
separates historical signature projection from an explicitly bound complete
current-release `java.se` module image.

## First Maven runtime-scope export row

`BuildSourceSet.runtimeClasspathEntries` now carries a separately hash-bound
runtime projection. A module's runtime-only dependency remains absent from its
main compile classpath, while downstream test compilation through a reactor edge
receives that runtime export. The permanent generated fixture compiles a local
API JAR, declares it runtime-only in an intermediate reactor module, and proves
that an acceptance-test module resolves it without Maven execution or network.
The row passed the full native matrix in run `29671357121`.

Optional read-only Magrathea qualification now reports `No diagnostics`; the
prior 11 SQLite/JDBC rows were the matching runtime-scope export defect.

## First Maven mediation row

The offline resolver now uses bounded breadth-first traversal. A nearer direct
version wins over an earlier transitive version, equal-depth conflicts retain
Maven declaration order, and effective root dependency management is applied
before identity selection. Permanent evidence also proves optional transitive
and explicitly excluded artifacts remain absent, while a direct `provided`
artifact remains owner-compile-visible without exporting its child. A selected
version exposes a method absent from the rejected version, so clean staged JDT
bindings validate the classpath rather than filenames alone. The mediation row
passed all four native platforms in run `29687396137`.

## Exact diagnostic identity row

Managed apply, recipe diagnostics and Java mutation deltas no longer identify an
error by code/message alone. Identity also binds severity, evidence, category,
location precision, normalized path and exact range. A permanent RED/GREEN gate
fixture proves that moving an otherwise identical compiler error is an introduced
regression and is refused before WAL creation or workspace mutation. Import
sorting acceptance uses complete source fixtures, so qualification does not rely
on range-shifted unresolved-type baselines.

## Bounded range and unsupported-variant row

A local Maven version range scans at most 1,024 version directories and selects
the highest matching version; JDT validates a method unique to that version.
Only `jar` and `test-jar` dependency types enter semantic classpaths. Arbitrary
artifact types and POM relocation metadata make the affected source set typed
unavailable and suppress derivative bindings rather than consuming a plausible
but incorrect binary. The range/refusal row passed all four native platforms in
run `29688995827`.

## First source-set availability row

Maven resolution records main and test missing artifacts independently and
projects `java.classpath.status/message` onto each `BuildSourceSet`. Default
Java diagnostics emit one typed root for an unavailable test source set,
suppress only derivative type/binding rows under that test root, and retain an
intentional unresolved main-source type. The RED fixture previously emitted one
module-level root and hid the genuine main error. The same row runs through the
packaged CLI, daemon and MCP with an exact read-only tree hash. The row passed
all four native platforms in run `29682355909`.

Availability is propagated transitively over source-set reactor edges. A missing
main compile environment marks dependent main/test sets unavailable, while an
unavailable runtime export marks only downstream test environments. Permanent
fixtures prove two propagated roots across a broken compile edge, one test root
across a missing runtime export, no derivative binding rows, and an independent
genuine main error that remains visible. The propagated-availability row passed
all four native platforms in run `29683959746`.

## Executable evidence

`JdtJavaSemanticAnalyzerTest` verifies that Java 8 authoritative analysis rejects
`List.of`, even when RefactorKit runs on JDK 21, while retaining core platform
bindings. It also verifies typed refusal for a source-only Maven configuration.
`JavaReleasePlatformAuthorityTest` covers deterministic identity, drift-sensitive
hashes, missing release signatures, malformed archives, bounded entry refusal and
unsupported inputs. `JavaAuthoritativeDiagnosticsAcceptanceTest` covers release
21 Java SE/transitive-reactor visibility, historical API rejection, module-result
filtering and concise unavailable roots.

`scripts/smoke-packaged-java-authoritative-diagnostics.py` runs CLI, daemon and
MCP with `JAVA_HOME` removed. It proves `java.sql`, `java.net.http` and sibling
reactor bindings for release 21; release 8 rejects `HttpClient` and `String.isBlank`
without fabricating Java 8 or reactor failures. The fixture hashes its tree to
prove diagnostics are read-only. Native packaged qualification passed on Linux
x86-64, Windows x86-64, macOS x86-64 and macOS ARM64 in run `29660037679`.
