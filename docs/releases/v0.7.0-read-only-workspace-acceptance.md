# v0.7.0 read-only workspace acceptance

Status: implementation acceptance for the fix to the `v0.6.2` read-only lock-file
defect. This is not a release tag.

## Defect

Immutable `v0.6.2` invokes `PatchEngine.recover()` during daemon `project.open`,
MCP `project_scan`, and LSP initialization. Recovery acquires the managed writer
lock and creates `.refactorkit/workspace.lock` even in a fresh workspace with no
journal. Those surfaces advertise no workspace write, so the metadata file is a
contract violation even though source bytes remain unchanged.

## Corrected boundary

Read-only startup now calls `PatchEngine.inspectRecovery()`:

- missing `.refactorkit/transactions` returns cleanly without creating a directory;
- journal and quarantine paths are inspected without lock acquisition;
- records are decoded without corruption quarantine or lifecycle updates;
- orphan temporary records are detected but not deleted;
- `PREPARED`, `APPLYING`, `ROLLING_BACK`, and `RECOVERY_REQUIRED` states produce
  `transaction.recoveryRequired` without changing source, journal, metadata, or
  lock bytes;
- completed `APPLIED`/`ROLLED_BACK` records do not block reads.

Recovery is now separately and explicitly authorized:

```bash
refactorkit patch recover --root /workspace
```

Daemon API `patch.recover` is `experimental`, `requiresProject=false`, and
`writesWorkspace=true`. It acquires the existing writer lock and retains the
qualified compensation behavior. Read-only project open refuses pending recovery
and tells the caller to invoke the explicit operation.

## Acceptance

```bash
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH="$JAVA_HOME/bin:$PATH"
./gradlew clean build goldenTest --no-daemon
./gradlew :modules:refactorkit-cli:refactorkitRuntimeZip \
  :modules:refactorkit-cli:smokePackagedCli --no-daemon
python3 scripts/smoke-packaged-typescript.py
python3 scripts/smoke-packaged-kotlin.py
python3 scripts/smoke-packaged-kill-recovery.py
```

Evidence requirements:

- 581 JVM tests and all 23 golden cases pass locally;
- fresh `PatchEngine.inspectRecovery()` creates no `.refactorkit` directory;
- interrupted journal inspection creates no `workspace.lock`, changes no journal
  state/bytes, and changes no source bytes;
- daemon `project.open`, MCP `project_scan`, and LSP initialization leave a fresh
  workspace without `.refactorkit`;
- packaged TypeScript start/search/definition/references/diagnostics and Kotlin
  diagnostics/symbols/definition hash every source and hidden metadata file before
  and after the complete read-only phase;
- an interrupted packaged 111-file transaction refuses ordinary open, succeeds
  through explicit `patch.recover`, then opens read-only with byte-identical source
  restoration and a `ROLLED_BACK` journal;
- native CI passes Linux x86-64, Windows x86-64, macOS x86-64 and macOS arm64.

The `v0.6.2` tag is immutable and remains affected. This correction applies only
to subsequent builds/releases.
