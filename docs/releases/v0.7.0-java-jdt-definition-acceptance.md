# v0.7.0 Java JDT Navigation Acceptance

Status: accepted experimental provider row.

## Qualified scope

`intelligence.query` supports `kind=definition` and `kind=references` for Java
saved-snapshot source authority. The request requires exact workspace snapshot, optional exact index
generation, workspace-relative path and zero-based UTF-16 position.

The provider:

- analyzes the exact `ProjectSnapshot` with Eclipse JDT 3.44 bindings;
- resolves declarations and bound usage locations without lexical fallback;
- returns zero or one exact definition location;
- returns up to 1,000 deterministically ordered references with optional
  declaration inclusion, total/returned counts, explicit truncation and
  warning-derived completeness;
- caches normalized analysis for two exact snapshots using session-owned LRU
  state;
- clears the cache across workspace sessions;
- checks cooperative cancellation between compilation units and before cache
  publication;
- refuses analyses above 10,000 Java source files or 256 MiB aggregate UTF-8 source;
- lazily contributes up to 50,000 bound symbols as
  `java-jdt-bindings-v1`, compiler evidence, semantic completeness;
- exposes only path-free deterministic provenance, never JDT binding keys;
- creates no `.refactorkit` metadata.

Saved-file reconciliation invalidates the JDT provider partition whenever Java or
non-source build evidence changes. The old cache entry may remain bounded but can
never answer a request for the new snapshot key.

## Verification

- full repository suite: 619 JVM tests, zero failures/errors/skips;
- repeated definition requests: one miss followed by one cache hit;
- usage `service.greet()` resolves to the exact `UserService.greet` declaration;
- the same position returns declaration plus usage when requested, or usage only
  when `includeDeclaration=false`;
- semantic partition appears in `index.status` only after qualified JDT analysis;
- packaged TypeScript and mutation acceptance remains green.
