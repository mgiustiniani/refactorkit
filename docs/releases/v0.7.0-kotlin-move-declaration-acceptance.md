# v0.7.0 Kotlin/JVM move-declaration acceptance

Status: import/consumer rows complete across all acceptance layers. Whole-file
movement with compiler-proven private top-level helpers has local and packaged
CLI/daemon/MCP acceptance; native qualification is pending.

## Qualified first row

`moveDeclaration` accepts one compiler-proven public top-level Kotlin/JVM type
that is the only top-level declaration in its file. The destination remains in
one authoritative non-generated source-root path. A supported Kotlin or Java
consumer either has one explicit non-aliased, non-star import, belongs to the
exact old package and resolves the type implicitly, or uses the exact old fully
qualified identity at every compiler-proven target token.

K2 proves the declaration and Kotlin usages. Ephemeral hash-attested K2 classes
let JDT prove Java uses of the same binary identity. Preview changes only the
package token, explicit import tokens, deterministic new imports for proven
same-package consumers, exact fully-qualified target identities and one
source-root-relative file path.

The staged snapshot must compile with K2, publish the destination binary identity
and resolve Java consumers through JDT. Preview never writes source, class, JAR
or transaction metadata into the workspace.

## Transaction evidence

Local library acceptance previews, applies and rolls back this mixed move:

```text
fixture.api.PublicGreeting
  -> fixture.api.v2.PublicGreeting

Kotlin import: fixture.api.PublicGreeting -> fixture.api.v2.PublicGreeting
Java import:   fixture.api.PublicGreeting -> fixture.api.v2.PublicGreeting
```

The declaration file, Kotlin import and Java import apply in one `PatchEngine`
transaction. Rollback restores the original file path and bytes.

Packaged acceptance exercises a read-only CLI preview plus daemon and standalone
MCP preview/apply/rollback for `PortableGreeting`. Semantic lease and snapshot
authority are required by both transports; daemon additionally binds the
workspace-index generation.

## Refusal evidence

Executable tests cover absent external-consumer approval, explicit imports,
same-package Kotlin/Java import insertion, exact fully-qualified Kotlin/Java rewrites, zero-consumer public types, independent Kotlin-only and Java-only consumer sets,
K2-proven alias uses with exact alias preservation, deterministic explicit
imports after preserved Kotlin/Java package-star imports, whole-file movement of
private top-level helper types/functions/properties, and refusal of mixed imported and qualified
uses. The implementation also refuses ambiguous ownership, generated roots,
destination conflicts, multiple top-level declarations, default packages,
dynamic/framework evidence and incomplete K2/JDT evidence.
