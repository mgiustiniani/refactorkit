# RefactorKit v0.5.0 Platform Release Plan

Status: active.

Baseline: published `v0.4.0`; main `0.5.0-SNAPSHOT`; API `0.2` until an explicit
migration is accepted.

## Objective

`v0.5.0` begins the production multi-language kernel and expands the
self-contained runtime from Linux-only publication to native Windows and macOS
artifacts. Cross-platform packaging is not considered complete until managed
preview/apply/WAL/recovery/rollback semantics pass natively, not merely until
`--help` starts.

## P1 — Native self-contained runtime matrix

Target assets:

| Platform | Runner/build architecture | Asset suffix |
|---|---|---|
| Linux | x86_64 | `linux-x86_64` |
| Windows | x86_64 | `windows-x86_64` |
| macOS | Intel x86_64 | `macos-x86_64` |
| macOS | Apple Silicon arm64 | `macos-aarch64` |

Requirements:

- build each `jlink` image on a native GitHub runner;
- include only the packaged runtime and never require global Java at execution;
- provide native shell/batch launchers and path-with-spaces/Unicode tests;
- run full build/tests plus packaged version, signed definition/reference,
  format/apply/rollback and source-restoration smoke on every platform;
- generate a platform checksum and SPDX JSON SBOM;
- publish build-provenance and SBOM attestations per runtime;
- aggregate all assets into one immutable GitHub Release;
- document PowerShell, Intel macOS and Apple Silicon installation/verification.

## P2 — Filesystem and transaction parity

Before Windows or macOS is marked supported for writes:

- introduce platform-aware durability capability/strategy where Java NIO differs;
- prove atomic replacement and the qualified directory durability contract;
- cover POSIX, ACL, DOS attributes and unsupported-view behavior;
- detect filesystem case sensitivity and reject colliding/case-only paths safely;
- cover Windows drive/UNC/long paths, junctions/reparse points and open-handle
  replacement behavior;
- cover macOS APFS, default case-insensitive volumes, ACL/xattr and symlinks;
- add native process-kill/restart, journal corruption, rollback conflict and
  recovery acceptance;
- preserve explicit refusal when a filesystem cannot meet the reported contract.

Packaging may be published as preview/read-only for a platform only if the asset
and README say so explicitly. Full support requires managed-write parity.

## P2A — Java 21 Maven reactor and ownership relocation

Status: complete on main. Normative decision: ADR 0010.

- [x] Embedded, plugin-free Maven effective-model discovery, offline by default.
- [x] Inherited Java 21, BOM/dependency-management, reactor edge, local artifact,
  and main/test/generated source-set analysis with hash-bound evidence.
- [x] Concise typed `buildModel.unavailable`, `classpath.unavailable`, and
  `sourceLevel.unavailable` roots instead of default classpath cascades.
- [x] `moveSourceRoot` rename-only preview/apply/WAL/rollback across Maven modules.
- [x] Committed `samples/java-maven-reactor-21` and read-only Magrathea acceptance.
- [x] Library, CLI, daemon, and MCP exposure with portable paths and typed
  relocation refusals.

Acceptance evidence:

- implementation commit `a368f54970597c6821bd1eacd17d37b7e9a2ae7c`;
- portable CLI follow-up `7edcf36c213cf928ec0dea6379aa4d7b66db1c32`;
- 433 unit/integration tests and 23/23 golden cases;
- packaged Magrathea scan: five modules, Java 21 effective model/classpath
  available in every module, zero diagnostics, cross-module references, and no
  workspace mutation;
- native CI run `29205094718`: Linux, Windows x86_64, macOS x86_64, and macOS
  arm64 green.

Completion of P2A does not by itself qualify the full `v0.5.0` platform release.
P1, P2, P2B, P3, and P4 retain independent gates.

## P2B — JVM build-model productionization

Status: complete on `main`; privileged provider implementation remains an explicit future capability, not a P2B release dependency.

- [x] Add internal language-neutral core contracts for `BuildModelProvider`,
  discovery policy, `BuildModel`, modules, source sets, scoped module edges,
  outputs, generated roots, typed model diagnostics, and status.
- [x] Bind build-model projections into `ProjectSnapshot` hashing while retaining
  API `0.2` `Module` compatibility.
- [x] Project current Java discovery through explicit `maven-effective-v1`,
  `gradle-declarative-v1`, and `java-conventional-v1` providers and expose
  redacted daemon/MCP summaries.
- [x] Migrate JDT parser environments (main/test roots, generated roots,
  classpaths, source levels, and transitive module visibility) to provider source
  sets, with compatibility fallback for snapshots without build models.
- [x] Migrate source ownership/planner consumers (`moveSourceRoot`, importer
  targeting, formatter selection, package diagnostics, and staged JDT overlays)
  to exact/longest-prefix Build Model queries, retaining fallback only for
  snapshots without build models.
- [x] Retain the SPI inside RefactorKit rather than creating another
  service/product.
- [x] Harden Maven variant mediation, classifier, `test-jar`, and `systemPath`
  scope/evidence coverage, including pre-WAL drift refusal.
- [x] Model effective Maven main/test source directories and active-profile
  declarative build-helper add-source roots without plugin execution, rejecting
  workspace/symlink escapes.
- [x] Define bounded provider-scoped explicit Maven profile activation and
  hash-bind selected IDs.
- [x] Require anonymous HTTPS Maven Central plus bounded mandatory SHA-256
  sidecars before atomic publication; distinguish pre-existing local artifact
  content evidence from repository authenticity.
- [x] Define the credential-safe future-provider boundary in ADR 0011: new
  provider identity, opaque scoped/expiring non-serializable handles, origin
  pinning, redaction, provenance, and no implicit Maven settings access. Current
  authenticated repository support remains deliberately unavailable.
- [x] Select a safe default Gradle strategy: bounded declarative parsing only,
  with no Wrapper/settings/script/task/plugin/Tooling API execution and explicit
  `PARTIAL`/`EXECUTION_REFUSED` status.
- [x] Model literal Gradle main/test/integration/custom source sets, generated
  outputs, Java levels, project edges, and scanner classpath evidence.
- [x] Define any future executable Gradle provider boundary in ADR 0011: new
  provider identity, isolated worker, explicit distribution provenance,
  environment/filesystem/network limits, timeout/cancellation/process-tree
  termination, schema validation, and hostile-workspace native acceptance.
- [x] Complete typed, bounded, redacted Build Model summary DTOs, capability
  flags, canonical available/partial/offline-missing/unavailable/execution-refused
  statuses, truncation metadata, and checked-in schema-key contract snapshots.
- [x] Add native packaged Maven reactor scan/diagnostics and `moveSourceRoot`
  preview/apply/WAL/rollback byte-identity acceptance with `JAVA_HOME` removed on
  every supported OS.
- [x] Prove case-folded relocation collision refusal on Windows and both macOS
  architectures (plus Linux) through the packaged runtime matrix.

P2B must not broaden generated-source write authority or permit build-model
providers to write outside `PatchEngine`.

## P3 — Distribution trust

Status: baseline complete; platform signing implementation remains explicit future work.

- [x] Evaluate macOS Developer ID signing/notarization and document current
  unsigned Gatekeeper/quarantine behavior without encouraging blind bypass.
- [x] Evaluate Windows Authenticode/SmartScreen behavior and document the current
  unknown-publisher state.
- [x] Keep ZIP publication and verification independent of installer/signing work.
- [x] Verify checksum, bounded safe layout, reproducible timestamps, executable
  permissions, embedded modules, native ELF/PE/Mach-O architecture, and packaged
  launcher execution on each native builder.
- [x] Re-download per-platform job artifacts in the publish job and independently
  verify checksums/layout/modules/architecture before release publication; retain
  platform SPDX SBOM and build/SBOM attestations per subject digest.

See [`distribution-trust.md`](../distribution-trust.md). Signing credentials,
notarization, Authenticode timestamping, rotation/revocation, and native signed-
asset acceptance are intentionally not claimed until separately implemented.

## P4 — Multi-language kernel

Status: in progress.

- [x] Add a bounded language adapter registry with canonical descriptors, unique
  language/extension ownership, deterministic mixed-language aggregation, and
  explicit selection/symbol/language routing with typed ambiguity refusal.
- [x] Extract the internal Build Model SPI and source-set/module graph from the
  P2A Maven reference implementation.
- [x] Generalize compiler/language-server/native-AST/structural/lexical evidence
  and fail closed when an adapter returns weaker evidence than stable authority.
- [x] Add a bounded external semantic-process lifecycle manager with cleared
  explicit environment, executable/argument provenance hashes, process capacity,
  stdout/stderr limits, natural-exit cleanup, cancellation, and graceful/forced
  process-tree termination.
- [x] Integrate the lifecycle manager with `ExternalLspAdapter`: byte-counted
  bounded LSP framing, initialization/version/capability provenance, request
  deadlines, cancellation, source-only workspace overlays, URI remapping, and
  overlay source-mutation detection.
- [x] Package real Tree-sitter JNI runtime plus TypeScript and JavaScript grammars
  for Linux x86_64, Windows x86_64, macOS x86_64, and macOS arm64; use bounded
  native parse trees for outlines and identifier search with heuristic fallback
  only for languages without a packaged grammar.
- [x] Parse both standard LSP WorkspaceEdit forms with strict schema/resource
  operation policy and normalize untrusted paths, ranges, overlaps, document
  versions, generated roots, symlinks, limits, and structural conflicts into
  non-authorized core proposals.
- [ ] Complete capability schemas, timeout/cancellation, process provenance and
  output/resource limits across integration surfaces.

No language adapter inherits stable mutation authority from platform packaging.

## Acceptance

`v0.5.0` requires green native matrix jobs, platform-specific managed transaction
acceptance, completed P2B build-model gates, native packaged Maven reactor and
source-root relocation acceptance, all existing 23+ golden cases, Linux
regression smoke, reproducible per-platform archives, release
asset/SBOM/attestation verification, updated
README/packaging/migration documentation, and no hidden global runtime dependency.
