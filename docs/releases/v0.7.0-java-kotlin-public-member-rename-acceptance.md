# v0.7.0 bidirectional Java/Kotlin public member rename acceptance

Status: local packaged acceptance complete; native qualification pending.

## Kotlin function used by Java

The first row accepts one public non-overloaded Kotlin function with one
compiler/class-file-proven JVM owner/name/descriptor and direct Kotlin plus Java
callers. K2 supplies exact Kotlin declaration/call tokens. The exact K2 overlay
classes let JDT resolve Java method binding tokens for the same owner and method.

## Java method used by Kotlin

The symmetric row accepts one public, non-overloaded, non-override JDT-bound Java
method. Bounded annotation-processing-free ECJ emits a hash-bound ephemeral JAR.
K2 FIR reports exact external callable uses by JVM owner and method name; JDT
supplies the Java declaration and Java calls. Staged ECJ/K2/JDT evidence must
resolve the renamed method.

Both rows refuse overload/bridge ambiguity, override or unsupported Kotlin
operator/infix shapes, callable references, generated sources, dirty evidence,
dynamic quoted names and public external-consumer risk unless explicitly
accepted.

## Transaction evidence

Daemon, CLI and MCP preserve snapshot/lease/index authority and select the
source-language-specific combined diagnostics gate. `PatchEngine` applies each
Java/Kotlin member rename in one WAL transaction; packaged acceptance rolls back
both source files byte exactly.
