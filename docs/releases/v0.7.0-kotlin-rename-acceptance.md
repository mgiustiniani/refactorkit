# v0.7.0 bounded managed Kotlin rename acceptance

Status: K4 private-declaration foundation accepted in GitHub Actions run
`29451134010`.

## Qualified boundary

The experimental compiler-backed row accepts non-local explicitly private Kotlin
classes/interfaces/objects/enum/annotation classes, non-overloaded direct-call
functions, direct field-backed properties, function value parameters and function
type parameters in one Kotlin-only JVM module. IDs derive respectively from JVM
type identity, callable owner/name/descriptor, field owner/name/descriptor, or
callable owner/descriptor/ordinal. K2 real PSI and resolved FIR symbols supply
exact UTF-16 declaration/reference tokens.

Java sources, public/external consumers, aliases, star imports, callable
references, delegated/constructor properties, operators/overrides, generated
sources, scripts/plugins, dynamic quoted names and incomplete/ambiguous evidence
return typed refusal. A Java source naming a public Kotlin type is covered by
`kotlin.renameCrossLanguageIncomplete`; no cross-language mutation is claimed.

## Managed transaction evidence

- preview requires exact snapshot, semantic lease and index generation;
- the post-image is simulated and compiled with the same attested K2 toolchain;
- introduced compiler errors refuse;
- apply revalidates authority and toolchain/build/classpath evidence under lock;
- `PatchEngine` writes WAL intent and owns all source writes;
- packaged acceptance applies and rolls back `InternalGreeting`, restoring source
  bytes exactly;
- shared packaged kill-during-write recovery validates interrupted APPLYING intent
  on every native platform in the same run;
- preview/read paths do not create workspace transaction metadata.
