# v0.7.x Workspace Index and Semantic Query Plan

Status: active development.

## Objective

Make RefactorKit the central, provider-neutral index and semantic read engine for
IDE, CLI, daemon, MCP and agent consumers. The IDE retains document interaction,
triggers and rendering; RefactorKit owns the attested workspace image, index,
provider routing and normalized optional JSON results.

This plan does not replace JDT, tsserver or K2. They remain semantic providers:

```text
IDE / CLI / MCP
    -> RefactorKit WorkspaceIndex + intelligence query
        -> Java/JDT
        -> TypeScript/tsserver
        -> Kotlin/K2 Analysis API
```

## W1 — Immutable workspace index foundation

- [x] Add a language-neutral `WorkspaceIndex` in core, keyed by exact
  `ProjectSnapshot.hash`.
- [x] Index every recognized source path, language, UTF-8 size, content hash,
  saved-document authority and all longest-root build provider/module/source-set
  ownerships.
- [x] Accept bounded provider symbol contributions with backend/evidence metadata,
  exact snapshot paths and deterministic ordering.
- [x] Reject stale-snapshot contributions, duplicate provider IDs, duplicate
  symbol entries, unsafe/missing source paths, invalid ranges and resource
  overflow.
- [x] Add a session-owned `WorkspaceIndexSession`; no global mutable index.
- [x] Reconcile a new snapshot by reporting added/modified/deleted/unchanged paths
  and retaining contributions only for unchanged source paths.

## W2 — First central query surface

- [x] Build the source inventory and Java declaration contribution during daemon
  `project.open` without workspace writes.
- [x] Add the typed core `SemanticQueryProvider` SPI, document authority,
  cancellation token, operation-specific request/response variants and provider
  attestation models. Unsupported variants remain representable without flattening.
- [x] Add read-only daemon `index.status` and additive API `0.2`
  `intelligence.query` for bounded workspace/document symbol search.
- [x] Correlate search with request ID, expected snapshot hash, index generation,
  provider metadata, total/returned counts and explicit truncation.
- [x] Add CLI `index [--json]` and
  `intelligence search --kind workspace-symbols [--json]`.
- [x] Merge qualified Kotlin compiler class contributions after the existing
  lease/snapshot-authorized symbol query.
- [x] Merge qualified TypeScript/JavaScript language-server declaration
  contributions after semantic startup/restart without weakening saved-snapshot,
  toolchain, build-model, process or semantic-lease authority. Projection is
  capped at 256 files, 50,000 symbols and one 30-second aggregate deadline;
  stop/close removes the provider partition. Acceptance:
  `v0.7.0-typescript-workspace-index-acceptance.md`.

## W3 — Incremental editor authority

- [ ] Add versioned immutable editor overlays to the common query contract.
  - [x] Add core `ImmutableEditorOverlay` validation/derivation, exact content and
    version authority, deterministic overlay hashing, provider snapshot creation,
    defensive limits and `SemanticQueryEnvelope` correlation.
  - [x] Migrate TypeScript `diagnostics.v2` overlay construction to the shared
    core model without changing API `0.2` wire compatibility.
  - [x] Parse common overlay authority on `intelligence.query` and route
    TypeScript/JavaScript `documentSymbols` against the derived provider snapshot,
    exact semantic lease and monotonic document versions. Temporary LSP documents
    are restored to saved content before returning.
- [ ] Reconcile file additions, modifications, deletions, build-model changes and
  provider/toolchain drift without rebuilding unrelated index partitions.
- [ ] Add bounded invalidation, cache eviction, file watching and stale-result
  rejection.
- [ ] Add asynchronous request scheduling, cooperative cancellation and query
  priority while preserving deterministic single-writer mutation semantics.

## W4 — Interactive semantic queries

Expose typed variants behind the same provider-neutral search/query facade:

- [x] completion: first TypeScript/JavaScript immutable-overlay row with bounded
  typed items, insertion/replacement range, optional additional edits, trigger
  context, explicit incompleteness, lease/version authority and restoration.
  Rich expected-type and visibility attestations remain later extensions;
- [x] hover: first TypeScript/JavaScript immutable-overlay row with bounded
  plaintext/Markdown sections, exact optional UTF-16 range, provider provenance,
  lease/version authority and saved-document restoration. Rich resolved identity
  remains a later extension;
- [x] signature help: first TypeScript/JavaScript immutable-overlay row with
  bounded overloads, exact parameter-label spans, active signature/parameter,
  trigger/retrigger context, lease/version authority and restoration;
- [ ] usage-location definition, references, document highlights and call/type
  hierarchy.

Every variant must have its own schema. A common endpoint must not flatten these
results into untyped strings. Unsupported providers return typed refusal; lexical
search never inherits compiler authority.

## W5 — Provider qualification

1. TypeScript/JavaScript through the existing persistent tsserver/LSP session.
2. Java through a cached incremental JDT environment.
3. Kotlin through a persistent bounded K2 Analysis API session.
4. Future adapters through the same query SPI and capability matrix.

Each provider must report snapshot/document authority, backend, evidence,
completeness, limits, truncation and known blind spots.

## Acceptance gates

- read-only open/index/query creates no `.refactorkit` metadata;
- deterministic index output across repeated scans and native platforms;
- exact stale snapshot and invalid path refusal;
- bounded source, provider, symbol, query and response sizes;
- no source content or toolchain paths leaked by status output;
- package smoke and native CI on Linux x86-64, Windows x86-64, macOS x86-64 and
  macOS arm64;
- completion/hover/signature capabilities remain refused per unsupported provider
  until real implementations, schemas, overlays, latency evidence and IDE
  acceptance pass.
