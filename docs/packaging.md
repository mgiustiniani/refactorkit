# Packaging

RefactorKit can be packaged as a self-contained CLI distribution with an embedded
Java runtime generated by `jlink`. The generated package does **not** require a
system Java installation at runtime.

## Tasks

From the repository root:

```bash
# Build unpacked self-contained CLI package
./gradlew packageCliRuntime

# Build zipped self-contained CLI package
./gradlew distCliRuntimeZip
```

Equivalent module-local tasks:

```bash
./gradlew :modules:refactorkit-cli:refactorkitRuntimeDist
./gradlew :modules:refactorkit-cli:refactorkitRuntimeZip
```

## Outputs

Unpacked package:

```text
modules/refactorkit-cli/build/package/refactorkit/
  bin/refactorkit
  bin/refactorkit.bat
  lib/*.jar
  runtime/bin/java
```

Zip package:

```text
modules/refactorkit-cli/build/distributions/refactorkit-runtime.zip
```

## Run packaged CLI

```bash
modules/refactorkit-cli/build/package/refactorkit/bin/refactorkit --help
modules/refactorkit-cli/build/package/refactorkit/bin/refactorkit scan samples/java-maven-simple
```

The launcher resolves its runtime relative to the package directory:

```text
$APP_HOME/runtime/bin/java
```

On Windows, use:

```text
bin\refactorkit.bat
```

## Runtime modules

Default `jlink` module set:

```text
java.base,java.logging,java.xml,jdk.unsupported
```

Override if needed:

```bash
./gradlew :modules:refactorkit-cli:jlinkRuntime \
  -Prefactorkit.runtime.modules=java.base,java.logging,java.xml,jdk.unsupported
```

## Notes

- The application bytecode still targets Java 8 where configured by the root build.
- The embedded runtime is built from the JDK used to run Gradle.
- `jlink` requires a JDK, not a JRE, at build time.
- The runtime package is OS-specific; build one package per target platform.
